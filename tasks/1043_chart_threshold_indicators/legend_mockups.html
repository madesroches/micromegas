Issue #1043 — what the legend shows when a series is colored by a SQL color column

Same underlying data in each card. Threshold = dashed crimson line. Recommendation: Option A.

RecommendedOption A — Data-aware swatch
query_a query_b

Legend matches the canvas. Palette color stays as identity + fallback (query_a, no color column → solid rust chip + rust line). A data-colored series shows a gradient chip sampled from its actual colors (query_b → green→red chip + gradient line). Works the same for single- and multi-series.

Option B — Palette = identity only
query_a query_b

Mismatch: the legend chip for query_b is blue (its assigned identity color) but nothing blue is drawn — the line is the green→red gradient. Smallest code change, but the legend no longer describes the canvas.

Option C — Single-series only
frame_time (ms)

Per-row color drives a single-query chart (the issue's core case — color bars by threshold). With 2+ queries, the chart keeps the palette identity colors and ignores the color column, so multi-series legends stay stable and readable. Simplest mental model, but no data-driven color in multi-series charts.
