What you can put in From / To

Each bound is resolved independently. An empty bound inherits that side of the screen range.

Inherit (default)

Both empty — the cell uses the screen range, exactly as today.

From(screen range)
To(screen range)
→ now-6h … now

Pinned relative range

Always the last hour, regardless of the screen range.

Fromnow-1h
Tonow
→ 15:20 … 16:20

Absolute window

A fixed incident window pinned to timestamps.

From2026-07-20T09:00Z
To2026-07-20T10:30Z
→ fixed 90-min window

One-sided override

Start from the screen's $from, end one hour earlier than usual.

From(screen range)
Tonow-1h
→ now-6h … now-1h

Derived from a variable

Range driven by a combobox / expression variable.

From$window_start
To$window_end
→ resolved from variable values

Derived from a selection

Zoom a downstream cell into a row/drag selection made upstream.

From$cell.selected.start
To$cell.selected.end
→ follows the selection live

Cell header indicator

A small clock chip on each query-backed cell header tells you at a glance whether it's on the screen range or an override. The chip always shows the resolved timestamps — never the raw macro — so you can debug exactly what window the cell queried. Hover to see the raw config ($cell.selected) it resolved from.

screen range 15:20:00 → 16:20:00 16:04:12 → 16:04:47
Muted = inherits screen range (no override) Gold = per-cell override active

Scope

Shown on query-backed cells: table, chart, log, propertytimeline, swimlane, transposed, flamegraph, map, perfettoexport, image, and variable (combobox / datasource). Hidden on markdown, referencetable, and horizontal-group cells. Distinct from Flame Graph's Initial From / To, which sets the interactive view range, not the query window.
