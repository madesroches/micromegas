Expand description

Builds a synthetic ExportLogsServiceRequest from one CloudWatch Logs DATA_MESSAGE: one Resource carrying logGroup/logStream/owner as identifying attributes, one LogRecord per logEvent (timestamp converted ms → ns, body = raw message, verbatim — CloudWatch does not parse message, so neither do we).

service.name = logGroup / service.instance.id = logStream so distinct log streams (distinct ECS tasks, Lambda instances, RDS instances) resolve to distinct process_ids via the existing process_id_from_resource formula — no CloudWatch-specific identity logic needed.

Public (rather than private) so tests/cloudwatch_logs_tests.rs can assert its shape directly, matching the build_webhook_request precedent in handler.rs.
