Map asset endpoints.

GLBs are stored gzipped under a .gz suffix on the configured prefix — the suffix is the only signal we need on the read path. Catalog listing filters for it, GET tacks .gz onto the requested name and always responds with Content-Encoding: gzip, and PUT gzips the body before writing. That keeps the design portable: no PutOptions::attributes, so object_store::LocalFileSystem (which rejects attributes with NotImplemented) works the same as S3/GCS.

Entry returned by the catalog endpoint. size is the on-disk (gzipped) byte count — that is what storage is billed against and what the GET response will set as Content-Length.

UploadResponse
200 response body for a successful upload. The frontend refetches the catalog after each upload, so the response only needs to confirm what was stored (filename + on-disk byte count); last_modified is owned by the object store and surfaces on the next catalog fetch.
